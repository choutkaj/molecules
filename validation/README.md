# Validation

Reference validation is separated from the pure Rust library.

- `validation/features/<feature-id>/<corpus-id>.toml` contains feature/corpus manifests and fixture provenance.
- `validation/external_sources/` contains compact externally supplied records pinned by URL and checksum.
- `validation/corpora/` describes optional large local datasets that are not committed to git.
- `validation/data/` is for local unpacked corpora and is ignored by git.
- `validation/reference/rdkit/` contains small-molecule reference generators.
- `validation/reference/biopython/` contains macromolecular reference generators.
- `validation/features/<feature-id>/golden/` contains normalized JSON expectations.
- `validation/status/` contains generated passing evidence written only by validation with `--update`.

Golden files should record the input, expected normalized behavior, reference implementation, and reference version.
For every fixture listed in a validation manifest, `cargo xtask validate --feature FEATURE_ID --corpus CORPUS_ID`
requires the corresponding `golden/<fixture-slug>.json` file and compares its `expected` payload
against normalized Rust implementation output.

Validation molecule fixtures must be externally supplied records, not hand-authored toy systems. Each
fixture listed in a corpus manifest must have a matching `fixture_sources` entry with a non-manual
source, HTTPS URL, and SHA-256. Hand-authored malformed inputs belong in unit tests, not in
reference-golden validation.

Named validation corpora are:

- `tiny`
- `pubchem-100`
- `pubchem-1000`
- `pl-rex`
- `enamine-diversity`
- `pdb-10`
- `pdb-100`

`tiny` is intentionally fast and narrow. A feature's `validation_required` list defines the evidence
needed for its overall dashboard check. Plain validation is read-only. Add `--update` to record each
passing feature/corpus comparison, synchronize overall `validated`, and regenerate the dashboard.
Molecular golden data must be generated by the declared reference software, currently RDKit or
Biopython.

Examples:

```bash
cargo xtask validate --feature io.smiles.parse --corpus tiny
cargo xtask validate --feature all --corpus pubchem-100
cargo xtask validate --feature io.smiles.parse --corpus all
cargo xtask validate --feature all --corpus all --update
```
