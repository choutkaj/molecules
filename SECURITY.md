# Security Policy

## Supported Versions

There is no supported release. Version `0.0.0` is experimental, unpublished,
and may change without compatibility notice.

## Reporting

Do not place sensitive structures, crash inputs, credentials, or exploit
details in a public issue. While this repository is private, report a
vulnerability privately to the repository owner. Before making the repository
public, enable GitHub private vulnerability reporting and update this file
with the confirmed reporting URL or security contact.

Include the affected commit, parser or algorithm, smallest safe reproducer,
observed resource use, and whether the input may be redistributed. Fuzz crash
artifacts remain ignored until their contents and provenance have been
reviewed.
